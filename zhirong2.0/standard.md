# 项目规范

### 为什么不用分支?
- 例如main.vue 河南国安和其他的不同，合并到主分支就不合适。

### 文件分类不明确
- 例：main.vue 不同的main存在home文件夹下混乱
- 解：建立main的文件夹存放不同的main路由，并更新各自项目的router

### 项目功能不明确
- 例1：功能通用----uploadAct 增加新的需求，在对应的api中添加接口名称。由于项目共用组件，如果其他项目没有此接口就会发生错误，所以需要每个项目都配置
- 例2：功能不通用----$tokenName 项目组件功能区分标示

- **这样就导致各个项目穿插，后期维护无法参照**
- 解：各个项目不同点的记录文档，记录在此
